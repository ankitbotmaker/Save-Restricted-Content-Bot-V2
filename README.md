📚 About This Branch

This branch is based on Pyrogram V2, offering enhanced stability and a forced login feature.

Users do not need to log in for public channels, but for public groups and private channels, login is required.

Scroll down to the Features section for more details.



---

🔧 Features

✅ Extract content from public & private channels/groups.
✅ Rename and forward content to other channels/users.
✅ Extract restricted content from other bots.

Use format: https://b/botusername(without @)/message_id (get from Plus Messenger).
✅ /login method along with session-based authentication.
✅ Custom captions and thumbnails.
✅ Auto-remove default video thumbnails.
✅ Delete or replace specific words in filenames & captions.
✅ Auto-pin messages if enabled.
✅ Download videos from 30+ sites (YT, Insta, Twitter, FB, etc.) using YTDLP.
✅ Login via phone number.
✅ Supports 4GB file uploads (file splitter for non-premium users).
✅ Enhanced Timer: Separate limits for free & paid users.
✅ Optimized looping for faster multi-file processing.
✅ Premium Access: Faster speeds & priority processing.
✅ Ad Setup: Shortlink ads & token system.
✅ Fast uploader via SpyLib (Telethon + Mautrix bridge).
✅ Upload directly to topics in topic-enabled groups.

⚠️ Secure Your Sensitive Variables

Never expose sensitive credentials like API_ID, API_HASH, or BOT_TOKEN publicly.
Use environment variables instead.

For VPS or Local Machine:

Edit config.py:

nano config.py

Or export variables:

export API_ID=your_api_id
export API_HASH=your_api_hash
export BOT_TOKEN=your_bot_token


For Cloud Platforms (Heroku, Railway, etc.):

Set variables directly in the platform’s dashboard.


Using a .env File:

Create a .env file and add:

API_ID=your_api_id
API_HASH=your_api_hash
BOT_TOKEN=your_bot_token

Important: Add .env to .gitignore to prevent exposure.


🔒 Why? If credentials are leaked, your bot can be hijacked. Always secure your environment variables.
